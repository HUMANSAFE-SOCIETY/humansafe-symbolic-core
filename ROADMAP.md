# HUMANSAFE ROADMAP

## Phase 0 — Foundation (COMPLETED)
- repository pubblico attivo
- licenza MIT
- prima release v0.1.0
- struttura base symbolic core

## Phase 1 — Stabilizzazione Core (IN CORSO)

Obiettivo: rendere il core funzionante e comprensibile

- fix parser (errori tipo ERR_MODULE_NOT_FOUND)
- struttura cartelle coerente
- dizionario simbolico funzionante
- test base del sistema
- primo flusso input → interpretazione → output

Output:
→ core eseguibile senza errori

---

## Phase 2 — Motore Interpretativo

Obiettivo: dare “intelligenza” al sistema

- interpretazione segnali utente
- stato interno (state machine)
- livelli di intensità/emergenza
- output coerenti (non casuali)

Output:
→ sistema che capisce cosa sta succedendo

---

## Phase 3 — HUMANSAFE SOS ENGINE

Obiettivo: sicurezza reale

- livello 1: auto-aiuto (riflessione)
- livello 2: contatti fidati
- livello 3: emergenza reale
- logica anti-spam
- conferme intenzionali

Output:
→ sistema che reagisce al bisogno umano

---

## Phase 4 — SENSORIUM (APP ANDROID)

Obiettivo: interfaccia reale

- app Expo React Native

- UI dark, futuristica
- collegamento al core

Output:
→ Humansafe utilizzabile su smartphone

## Phase 5 — Ecosistema Humansafe

Output:
→ piattaforma viva
